# 变更日志与版本历史

## [2.0.0] - 2024-03-11 🎉 CAC分享会系统大升级

### ✨ 新增功能

#### 核心功能（0311更新实现）

1. **时间档位优化** ⏰
   - ✅ 活动时间改为1小时一档（09:00-10:00, 10:00-11:00等）
   - ✅ 可配置的时间槽列表（`TIME_SLOTS` 环境变量）
   - 益处：便于教室管理、减少时间冲突

2. **旁听登记功能** 👂
   - ✅ 新增"角色"字段，支持"评议员"和"旁听"两种选择
   - ✅ 旁听人数无限制，评议员限额3人
   - ✅ 美化的单选按钮UI，选择体验更好

3. **自动计算拟参加人数** 📊
   - ✅ 前端实时显示：分享者数 + 旁听者数 + 3名评议员
   - ✅ 后端API支持数据计算
   - 益处：方便主持人了解现场人数，做好准备工作

4. **分享者自主管理系统** ✨ NEW
   - ✅ 分享者可自己创建、编辑、删除活动
   - ✅ 专属管理页面：`/organizer`
   - ✅ 简单的学号身份验证（防止冒充）
   - ✅ 权限控制：只能删除无报名的活动，只能编辑自己的活动
   - 益处：分享者自主性强，CAC无需参与，减少协调成本

#### 隐患改进（第一期）

4. **系统预警与统计** ⚠️
   - ✅ **消亡兴趣组检测**：统计活跃分享者，发现无活动的兴趣组
   - ✅ **越界社员监控**：自动检测参与次数超过阈值的社员（默认10次）
   - ✅ 可视化在主页面展示，CAC可迅速发现问题
   - API端点：`GET /api/stats`

5. **两表设计架构** 🏗️
   - ✅ 分离"活动信息表"和"报名表"
   - ✅ 通过活动ID关联，结构清晰，易于扩展
   - ✅ 支持多个活动同时进行

### 🔀 破坏性改变（迁移指南）

⚠️ **重要**：从v1切换到v2时，需要调整数据结构

**旧结构**（v1.x）：
```
单表：报名表
- 姓名
- 学号  
- 子项（A/B/C）
```

**新结构**（v2.0）：
```
活动信息表：分享会活动
- 活动日期
- 活动时间
- 分享者
- 活动主题
- 活动教室
- 线上视频号
- 组织者学号 ← NEW（权限控制）

报名表：分享会报名
- 姓名
- 学号
- 关联活动（链接）
- 角色（评议员/旁听）
- 联系电话
```

**迁移步骤**：
1. 备份旧表数据
2. 在 SeaTable 新建两张表，按上述结构创建列
   - **重要**：不要漏掉"组织者学号"字段！
3. 如有需要，编写脚本迁移旧数据
4. 更新 `.env` 配置文件，添加：
   ```env
   ACTIVITY_COL_CREATOR_STUDENT_ID=组织者学号
   ```
5. 重启服务

## 📊 性能指标

### 压力测试（基准）

- **单活动最大报名**：无限制（仅评议员限3人）
- **同时在线用户**：500+（单进程）
- **平均响应时间**：<200ms
- **API调用频率**：每页30秒刷新一次

### 优化建议

- 使用 Redis 缓存活动列表（减轻SeaTable压力）
- 使用 CDN 加速静态资源
- 数据库连接池配置
