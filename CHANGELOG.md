# Changelog

本项目版本记录遵循“可读、可追溯、面向发布”的原则。

## [Unreleased]

### Added

- 首页新增“新手必看：30 秒上手”规则说明模块，降低新成员使用门槛
- 治理看板新增时间冲突检测面板
- 治理看板新增评议文档评分面板（支持就地评分）
- 新增接口 `GET /api/reviewer-submitted-docs`，返回可评分的已提交评议文档
- 新增 MySQL 数据后端（`DB_BACKEND=mysql`）与自动建表能力
- 新增迁移脚本 `scripts/migrate_seatable_to_mysql.py`，用于历史数据导入

### Changed

- 管理端治理接口 `GET /api/admin/dashboard` 增加 `time_conflicts` 聚合字段
- 首页信息架构优化，规则说明前置
- 项目文档体系重构：删除阶段性遗留文档，保留核心运行文档
- 数据层默认由 SeaTable 切换为 MySQL，减少远程表服务带来的卡顿
- 活动/报名/评分统计增加聚合缓存，降低大数据量导出与列表查询耗时

### Fixed

- 修复治理看板对冲突数据缺乏展示的问题
- 修复“兴趣组健康度为空”场景下的验收误判（实际为源数据为空）

## [2026-03] - 当前可交付基线

### 功能覆盖

- 三入口：`/`、`/organizer`、`/profile`
- 活动全流程：创建、报名、评议、结项、治理统计
- 组织治理：月报、组健康、评议预警、边界预警、冲突检测
- 个人工作台：任务聚合与推荐活动

### 文档

- `README.md`：项目总览、架构、API、部署入口
- `SETUP_GUIDE.md`：从 0 到上线的操作手册
- `CHANGELOG.md`：版本与变更记录
